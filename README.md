# De-Influencer SMS

## Goal

Help people second-guess impulse buys by SMS: they text the app (via **Twilio**), optionally send a **photo** of something they might buy, and get back a short, humorous **de-influencer** style roast generated by **Google Gemini**—plus a practical nudge (borrow or rent).

## Stack

- [Next.js](https://nextjs.org/) 14 (App Router)
- [Google Generative AI](https://ai.google.dev/) (`gemini-2.5-flash`) for vision + text
- [Twilio](https://www.twilio.com/) for inbound SMS (webhook expects Twilio-style form posts and returns TwiML)

## Setup

```bash
npm install
```

Create **`.env.local`** in the project root (next to `package.json`). This file is gitignored—do not commit secrets.

```bash
GEMINI_API_KEY=your_key_here
```

Get a key from [Google AI Studio](https://aistudio.google.com/).

## Run locally

```bash
npm run dev
```

Open the URL Next prints (often `http://localhost:3000`). If port 3000 is busy, Next uses **3001** (or another free port)—use that port in the URLs below.

## Check Gemini

With the dev server running:

```bash
curl -sS http://localhost:3000/api/health/gemini
```

Replace `3000` with your port if needed. A working key returns JSON with `"ok": true`.

## Test the SMS webhook (no Twilio required)

The webhook accepts `application/x-www-form-urlencoded` body fields like Twilio sends (`From`, `Body`, optional `MediaUrl0`). Example with a public test image:

```bash
curl -X POST http://localhost:3000/api/webhook \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "From=+15551234567" \
  -d "To=+15557654321" \
  -d "MediaUrl0=https://images.unsplash.com/photo-1526170375885-4d8ecf77b99f?auto=format&fit=crop&w=800&q=60"
```

Response is **TwiML** XML with a `<Message>` body. Omit `MediaUrl0` to exercise text-only conversation mode for that `From` number.

## Twilio in production

Point your Twilio phone number’s **Messaging webhook** to your deployed URL, e.g. `https://your-domain.com/api/webhook` (POST). Set the same **`GEMINI_API_KEY`** in your host’s environment variables (e.g. Vercel project settings), not in the repo.
